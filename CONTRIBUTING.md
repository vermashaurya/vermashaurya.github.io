## Contributing

Thanks for your interest in contributing! 🎉

This is a personal portfolio project, so contributions are not actively sought, but suggestions, improvements, and bug reports are always welcome.

---

### Ways to Contribute

- Report bugs or issues  
- Suggest improvements (UI, performance, accessibility)  
- Fix small issues via pull requests  

---

### Getting Started

1. Fork the repository  
2. Clone your fork:
   ```bash
   git clone https://github.com/your-username/vermashaurya.github.io.git
   ```
3. Create a new branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```

---

### Development

No build setup required — this is a static site.

To run locally:
```bash
open index.html
```

Or use a local server:
```bash
npx serve .
```

---

### Code Style

- Keep HTML semantic and accessible  
- Use existing CSS variables and design tokens  
- Maintain responsiveness across screen sizes  
- Keep JavaScript lightweight and dependency-free  

---

### Testing

Run Playwright tests before submitting changes:

```bash
npm install
npx playwright test
```

Ensure all tests pass before opening a PR.

---

### Pull Request Guidelines

- Keep changes focused and minimal  
- Write clear commit messages  
- Add screenshots for UI changes (if applicable)  

---

### Note

Since this is a personal portfolio, major design or structural changes may not be accepted.  
Feel free to open an issue first to discuss ideas.

---

Thanks again for stopping by! 🙂
